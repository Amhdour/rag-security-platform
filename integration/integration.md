# Integration Contract

**Implemented:** This document defines how `rag-security-platform` exchanges data with adjacent repositories in the AI Trust & Security Readiness system.

## PRODUCES (from this repository)

This repository produces evaluation and evidence artifacts generated by `integration-adapter`.

Primary produced outputs:
- Security evidence records aligned to `artifacts_schema/schema.json`
- Audit event logs
- Replay artifacts
- Eval result artifacts
- Launch Gate verdict artifacts

Example produced paths:
- `integration-adapter/artifacts/logs/audit.jsonl`
- `integration-adapter/artifacts/logs/replay/<run>.replay.json`
- `integration-adapter/artifacts/logs/evals/<suite>.jsonl`
- `integration-adapter/artifacts/logs/launch_gate/security-readiness-<timestamp>.json`
- `artifacts_schema/schema.json`
- `artifacts_schema/example_artifact.json`

## CONSUMES (into this repository)

This repository consumes runtime-aligned and governance-context data as read-only inputs.

Primary consumed inputs:
- Runtime/exporter payloads from Onyx-aligned sources
- Contract fixtures for test/evaluation runs
- Existing artifact bundles for verification and health checks

Example consumed paths:
- `integration-adapter/tests/fixtures/onyx_contracts/*.json`
- `integration-adapter/tests/fixtures/adversarial/**/*.json`
- `integration-adapter/artifacts/logs/*` (for verification/reporting commands)

## File formats used

- **JSON Schema**: shared contract definition (`artifacts_schema/schema.json`)
- **JSON**: inventories, summaries, launch verdicts, replay artifacts
- **JSONL**: audit trails and eval result streams
- **Markdown**: human-readable reports and bridge verdicts
- **Log text**: command output and health/report summaries

## Shared evidence payload contract

**Implemented:** Evidence payloads exchanged across evaluation/observability/presentation consumers should conform to `artifacts_schema/schema.json`.

Required fields:
- `event_type`
- `control_name`
- `threat_type`
- `stage` (`input|retrieval|context|model|tool|output|logging`)
- `decision` (`allow|deny|warn`)
- `reason`
- `timestamp`
- `request_id`
- `severity`

### Example payload

```json
{
  "event_type": "policy.decision",
  "control_name": "prompt_injection_guardrail",
  "threat_type": "prompt_injection",
  "stage": "input",
  "decision": "deny",
  "reason": "Detected instruction override pattern matching disallowed tool escalation instructions.",
  "timestamp": "2026-03-18T10:15:30Z",
  "request_id": "req-2f91f5f8",
  "severity": "high"
}
```

## Consumer mapping

- Evaluation workflows (this repo): read/write artifacts for scoring and gating.
- Observability/dashboard repo: reads artifacts as evidence inputs (read-only).
- Website/presentation repo: reads summarized/curated evidence outputs.

Unconfirmed: production deployment transport/channel details between repos are environment-specific and not validated in this workspace.


## Misplaced components and ownership boundaries

- **Implementation repo responsibility (not owned here):** runtime feature delivery and control enforcement logic.
- **Dashboard repo responsibility (not owned here):** UI/API observability behavior and dashboard endpoint evolution.
- **This repo responsibility (owned):** evaluation evidence generation, normalization, and Launch Gate readiness outputs.

**Implemented:** In this workspace, `onyx-main/` and `myStarterKit-maindashb-main/` are retained as read-only mirrors for compatibility/testing context, not active ownership domains.
